[다다익선의 역설이란?]


1. 다다익선의 역설
많은 사람들은 1개보다 2~3개를 어필하는 것을 더 좋다고 생각한다.
하지만 애매하게 여러개 쓰는 것보다 안 쓰는게 낫다. 애매한 내용 여러개를 쓰면 오히려 마이너스 되는 경우가 많기 때문이다.

- A 지원자
Java / Spring Boot
SQL / MySQL / MariaDB
HTML/CSS를 활용한 웹 제작
정보처리기사 자격증
컴퓨터 활용능력 자격증
...

-> 당연한 것들 나열 : 어필할게 없어서 주저리 주저리 적은 건가?

- B 지원자
문제점)
Spring Boot 서버에서 상품 리스트를 불러오는 로직에서 성능 저하 발견

해결 과정)
실행 계획을 기반으로 성능 저하의 원인 파악 후, 인덱스 생성을 통해 풀 테이블 스캔에서 인덱스 스캔을 하게끔 성능 개선

성과)
1000ms에서 50ms로 20배 성능 향상

-> 데이터베이스 성능 개선이라는 확실한 1가지만 어필 : 이 정도 깊이면 다른 역량도 뛰어나겠지? 만약 뛰어나지 않아도 저 정도 깊이로 금방 할 수 있는, 성장 가능성 있는 개발자구나?

# 다다익선의 역설 : 애매한 스펙 여러개 보다 임펙트 있는 스펙 1~3개를 제대로 구체적으로 어필하는게 낫다.

- 불필요하고 어필되지 않는 내용 삭제
- 채용공고의 자격요건이나 우대사항에 해당되지 않는 내용 과감히 삭제
- 애매한 것들은 최소한의 분량으로 제일 하단부에 작게 작성
